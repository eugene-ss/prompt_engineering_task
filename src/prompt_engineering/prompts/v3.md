**CATEGORY DEFINITIONS:**

**invalid_email** (field: Email):
- Must have exactly one @ symbol
- Non-empty parts before and after @
- Domain after @ must contain at least one dot
- Not empty/whitespace/NULL
- INVALID: "user@gmail" (no .com), "userexample.com" (no @), "" (empty)

**inconsistent_date** (field: Signup_Date):
- Flag if NOT in exact YYYY-MM-DD format
- INVALID: "07/15/2023",
 "2023/07/15", "15-07-2023"
- VALID: "2023-07-15"

**invalid_date** (field: Signup_Date):
- Apply ONLY to dates already in YYYY-MM-DD format
- Flag impossible calendar dates
- 2023 is NOT a leap year (no Feb 29)
- INVALID: "2023-02-29", "2023-13-01", "2023-11-31"

**missing_value** (any field):
- Empty string, whitespace only, "NULL" (any case), "None"

**negative_amount** (field: Purchase_Amt):
- Numeric value < 0
- INVALID: "-50.00", "-1.23"

**unrealistic_amount** (field: Purchase_Amt):
- Contains non-numeric characters OR > 50000
- INVALID: "1,500", "\"2000\"", "abc", "75000.00"

**age_outlier** (field: Age):
- < 0 or > 120
- INVALID: "-5", "150", "200"

**status_inconsistency** (field: Status):
- Not exactly "Active", "Pending", or "Cancelled"
- INVALID: "active", "PENDING", "Canceled", "cancelled"
