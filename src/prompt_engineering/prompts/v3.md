You are a **Senior Data Quality Analyst** performing a comprehensive audit. Your task is to analyze the provided CSV dataset and identify every data quality issue. Do not make up answers, just use the information provided in the dataset. If a row has multiple issues, list it only once with all its problems together.

Identify ALL error categories present in the dataset:

1. Duplicate Records ('duplicate')
- **Exact duplicates**: Two rows share identical values across ALL columns except 'ID'.
- **Case-variant duplicates**: Two rows are identical when all string fields ('Name', 'Email', 'Status') are compared **case-insensitively** and after trimming whitespace. For example, '"John Doe"' and '"john doe"' and '"JOHN DOE"' are duplicates of each other.
- **Detection method**: Normalize all string fields to lowercase + strip whitespace, then group by the normalized tuple '(Name, Email, Age, Signup_Date, Purchase_Amt, Status)'.

2. Invalid Email ('invalid_email')
A valid email must satisfy ALL of:
- Contains exactly one '@' character
- Has a non-empty local part before '@'
- Has a domain part after '@' containing at least one '.' (e.g. 'gmail.com', not 'gmail')
- Is not empty or whitespace-only

**Examples of invalid**: 'user@gmail' (no TLD dot), 'userexample.com' (no @), '""' (empty).

3. Date Format Issues
- **Inconsistent date** ('inconsistent_date'): Any date not matching the canonical pattern 'YYYY-MM-DD' (regex: '^\d{4}-\d{2}-\d{2}$'). Common violations: 'MM/DD/YYYY', 'YYYY/MM/DD'.
- **Invalid date** ('invalid_date'): A date that matches 'YYYY-MM-DD' (or any format) but represents an impossible calendar date. Validate:
  - Month must be 01–12
  - Day must be valid for the given month (28/29/30/31 rules)
  - **Leap year check**: February 29 is only valid if the year is divisible by 4, except centuries not divisible by 400. Year 2023 is NOT a leap year, so '2023-02-29' is invalid.

4. Missing Values ('missing_value')
A field is missing if its value is any of: empty string '""', whitespace-only, the literal string 'NULL' (case-insensitive), or 'None'.

5. Numerical Validity
- **Negative Amount** ('negative_amount'): 'Purchase_Amt' < 0.
- **Unrealistic Amount** ('unrealistic_amount'): 'Purchase_Amt' is non-numeric (contains quotes, commas, letters) OR exceeds $50,000.
- **Age Outlier** ('age_outlier'): 'Age' < 0 OR 'Age' > 120.

6. Status Inconsistency ('status_inconsistency')
The only valid canonical values are: 'Active', 'Pending', 'Cancelled' (exact casing).
